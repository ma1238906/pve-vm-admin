背景： PVE虚拟机管理功能强大且复杂，想要入手需要一定的学习；创建/管理虚拟机需要管理员手动在PVE-WebUI操作，不够简便。

1. 角色设定： 你是一位精通 Proxmox VE、Linux 系统集成及全栈开发的私有云架构师。

2. 定制化开发需求（弃用 PVE 原生 UI，达到简便创建管理虚拟机）：

后端开发（基于python语音）： 

基于 PVE API 开发。要求实现基于 VM Template 的快速克隆逻辑。（虚拟机模板提前由管理员手动在PVE-WebUI制作）

管理后台（Admin）：

    虚拟机管理：虚拟机创建、删除、修改配置、查询状态。

    用户管理（增删改查、内网独立鉴权）。

    资源分配：指定虚拟机给特定用户，设定 CPU/内存/磁盘配额。

用户门户（User Portal）：

    极简界面：仅显示分配给该用户的虚拟机。

    控制功能：开关机、重启。

    核心功能： 通过浏览器展示 noVNC 终端。处理 PVE Ticket 鉴权及 Websocket 代理转发。

前端开发（基于vue框架）：

侧重于“资源监控”与“批量操作”。管理员需要一眼看到整个物理节点的负载情况，并能快速分配桌面。用户界面极致简约。

管理员门户：

    仪表盘 (Dashboard)： 
    
        使用环形图显示物理服务器的 CPU 负载、内存占用、存储剩余。显示当前在线用户数与运行中的 VM 总数。

    虚拟机模板管理：

        展示虚拟机模板信息

    虚拟机管理 (VM Management)：

        表格视图：包含 VM ID、名称、所属用户、IP 地址（从 PVE 代理获取）、当前状态、操作栏。

        操作栏：克隆、迁移、修改配置（热插拔 CPU/内存）、强制关机、删除。

    资源配额与用户关联：

        创建引导：选择“模板” -> “分配给用户” -> “定义规格(cpu、内存、硬盘)”。

    日志审计：记录用户登录时间、开关机操作历史。

用户门户：

    我的桌面：

        采用 卡片式布局。

        卡片内容：系统图标（Windows/Ubuntu）、VM 名称、运行状态（呼吸灯效果）、运行时长。

    交互动作：

        连接：点击后跳转或弹出 noVNC 窗口。

    控制按钮：
        
        仅保留“开机”、“重启”、“关机”。
